### requestSongUnlock Server Function

**File:** `src/lib/server/billing.functions.ts`

```typescript
const RequestSongUnlockSchema = z.object({
	songIds: z.array(z.string().uuid()).min(1).max(500),
});

export type RequestSongUnlockResponse =
	| {
			success: true;
			newlyUnlockedIds: string[];
			alreadyUnlockedIds: string[];
			remainingBalance: number;
	  }
	| {
			success: false;
			error: "insufficient_balance";
			required: number;
			available: number;
	  }
	| { success: false; error: "invalid_songs"; songIds: string[] }
	| { success: false; error: "unlimited_access_active" }
	| { success: false; error: "internal_error" };

export const requestSongUnlock = createServerFn({ method: "POST" })
	.middleware([authMiddleware])
	.inputValidator((data) => RequestSongUnlockSchema.parse(data))
	.handler(async ({ data, context }): Promise<RequestSongUnlockResponse> => {
		const supabase = createAdminSupabaseClient();
		const result = await orchestrateUnlock(
			supabase,
			context.session.accountId,
			data.songIds,
		);

		if (Result.isError(result)) {
			const err = result.error;
			switch (err.kind) {
				case "insufficient_balance":
					return {
						success: false,
						error: "insufficient_balance",
						required: err.required,
						available: err.available,
					};
				case "invalid_songs":
					return {
						success: false,
						error: "invalid_songs",
						songIds: err.songIds,
					};
				case "unlimited_access_active":
					return { success: false, error: "unlimited_access_active" };
				case "db_error":
					return { success: false, error: "internal_error" };
			}
		}

		return {
			success: true,
			newlyUnlockedIds: result.value.newlyUnlockedIds,
			alreadyUnlockedIds: result.value.alreadyUnlockedIds,
			remainingBalance: result.value.remainingBalance,
		};
	});
```

**Key constraints:**
- Max 500 song IDs per request
- Input validation via Zod
- All-or-nothing semantics (orchestrated at `unlocks.ts` level)
- Returns typed error discriminator (not thrown exceptions)

### Unlock Orchestration (requestSongUnlock function)

**File:** `src/lib/domains/billing/unlocks.ts`

```typescript
export async function requestSongUnlock(
	supabase: AdminSupabaseClient,
	accountId: string,
	songIds: string[],
): Promise<Result<RequestSongUnlockResult, UnlockError>> {
	// 1. Read billing state
	const billingResult = await readBillingState(supabase, accountId);
	if (Result.isError(billingResult)) {
		return Result.err({ kind: "db_error", cause: billingResult.error });
	}

	// 2. Check unlimited access
	if (hasUnlimitedAccess(billingState)) {
		return Result.err({ kind: "unlimited_access_active" });
	}

	// 3. Call RPC
	const { data, error } = await supabase.rpc("unlock_songs_for_account", {
		p_account_id: accountId,
		p_song_ids: songIds,
	});

	// 4. Parse error responses
	if (error) {
		if (error.message.includes("insufficient balance")) {
			return Result.err({
				kind: "insufficient_balance",
				required: songIds.length,
				available: billingState.creditBalance,
			});
		}
		if (error.message.includes("not currently liked")) {
			return Result.err({ kind: "invalid_songs", songIds });
		}
		return Result.err({ kind: "db_error", cause: new DatabaseError(...) });
	}

	// 5. Emit control-plane change
	if (newlyUnlockedIds.length > 0) {
		try {
			await applyLibraryProcessingChange(
				BillingChanges.songsUnlocked(accountId, newlyUnlockedIds),
			);
		} catch (err) {
			console.error("[unlocks] Failed to apply library processing change:", err);
		}
	}

	const remainingBalance = billingState.creditBalance - newlyUnlockedIds.length;
	return Result.ok({ newlyUnlockedIds, alreadyUnlockedIds, remainingBalance });
}
```

**Key:** 
- Already integrated with TanStack Start server functions via `billing.functions.ts`
- Returns structured success/error responses (no thrown exceptions)
- Emits `BillingChanges.songsUnlocked` for control-plane processing

## Constraints (Frozen from DECISIONS.md & TERMINOLOGY.md)

### Product Invariants

- **All-or-nothing unlock** (DECISIONS.md §7): No partial fulfillment. If balance insufficient, entire request fails.
- **Max 500 song IDs per request** (DECISIONS.md §7): Validated at server function level.
- **Duplicate song IDs deduped silently** (DECISIONS.md §7): RPC handles deduplication.
- **Already-unlocked songs returned separately** (DECISIONS.md §7): `alreadyUnlockedIds` and `newlyUnlockedIds` separate in response; only net-new deduct balance.
- **Insufficient balance returns typed error** (DECISIONS.md §7): Not thrown exception; discriminated error response.

### User Visibility

- **Pack users only** (story scope): Show selection UI only when:
  - `hasUnlimitedAccess(billingState) === false` (no subscription/self_hosted)
  - `billingState.creditBalance > 0` (has purchased balance)
  - `billingState.plan === "free"` (not on subscription)
  
- **Hidden for unlimited** (story scope): Accounts with `unlimitedAccess.kind !== "none"` do not see selection UI.
- **Hidden for provider-disabled** (story scope): Self-hosted accounts (when `BILLING_ENABLED=false`) have `unlimitedAccess.kind === "self_hosted"` and should not see pack selection UI.

### Terminology (Frozen)

- Internal: **unlock** (verb), `creditBalance`, `SongDisplayState`
- User-facing: **Unlock** (action button), "songs to explore", "lock icon"
- Display state values: `locked | pending | analyzing | analyzed | failed` (TERMINOLOGY.md §2)
- Source values: `free_auto | pack | unlimited | self_hosted | admin` (TERMINOLOGY.md §2)
