### useActiveJobCompletionEffects Hook

**File:** `src/lib/hooks/useActiveJobs.ts`

Currently handles invalidation when enrichment or match snapshot jobs complete:

```typescript
export function useActiveJobCompletionEffects(
	accountId: string,
	enabled = true,
) {
	const { data } = useQuery(activeJobsQueryOptions(accountId, enabled));
	const queryClient = useQueryClient();
	const prevRef = useRef<ActiveJobs | undefined>(undefined);

	useEffect(() => {
		const prev = prevRef.current;
		prevRef.current = data;

		if (!prev || !data) return;

		if (prev.enrichment && !data.enrichment) {
			queryClient.invalidateQueries({
				queryKey: dashboardKeys.pageData(accountId),
			});
			queryClient.invalidateQueries({
				queryKey: dashboardKeys.stats(accountId),
			});
			queryClient.invalidateQueries({
				queryKey: likedSongsKeys.stats(accountId),
			});
			queryClient.invalidateQueries({
				queryKey: likedSongsKeys.all,
			});
		}
		// ... match snapshot handling ...
	}, [data, accountId, queryClient]);
}
```

**Key:** This hook is called in `AuthenticatedLayout` and tracks active job completion. It may need to handle billing-triggered job changes (though unclear if billing changes trigger jobs directly).

const handleCheckout = async (offer: CheckoutTarget) => {

### Display Helpers

```typescript
export function getPlanLabel(state: BillingState): string {
	if (state.unlimitedAccess.kind === "self_hosted") {
		return "Unlimited";
	}
	if (state.unlimitedAccess.kind === "subscription") {
		return state.plan === "quarterly" ? "3-Month Unlimited" : "Backstage Pass";
	}
	if (state.creditBalance > 0) {
		return "Song Pack";
	}
	return "Free Plan";
}

export function getDisplayBalance(state: BillingState): number | null {
	if (hasUnlimitedAccess(state)) {
		return null;
	}
	if (state.creditBalance > 0) {
		return state.creditBalance;
	}
	return null;
}
```

**Key:** These helpers already determine when balance should be displayed (pack users with positive balance only).

### Billing State Type

**File:** `src/lib/domains/billing/state.ts`

```typescript
export interface BillingState {
	plan: BillingPlan;
	creditBalance: number;
	subscriptionStatus: NormalizedSubscriptionStatus;
	cancelAtPeriodEnd: boolean;
	unlimitedAccess: UnlimitedAccess;
	queueBand: "low" | "standard" | "priority";
}

export type UnlimitedAccess =
	| { kind: "none" }
	| { kind: "subscription" }
	| { kind: "self_hosted" };

export function hasUnlimitedAccess(state: BillingState): boolean {
	return state.unlimitedAccess.kind !== "none";
}
```

## Constraints (Frozen from DECISIONS.md & TERMINOLOGY.md)

### Offer IDs
- **Canonical internal offer IDs:** `song_pack_500`, `unlimited_quarterly`, `unlimited_yearly` (TERMINOLOGY.md §2, §5)
- Never send Stripe price IDs to UI; only internal offer IDs (DECISIONS.md §2)
- `checkoutAttemptId` is part of signed request body for idempotency (DECISIONS.md §2)

### Pack Purchase Rules
- **Unavailable while unlimited active:** Pack purchase entry points (UI buttons/CTAs) must be hidden or disabled when `hasUnlimitedAccess(billingState) === true` (DECISIONS.md §1)
- **Provider-enabled only:** Hidden entirely when `BILLING_ENABLED=false` (story scope)
- **Single offer:** Only `song_pack_500` (500 credits + 25 bonus unlocks)

### Unlimited Upgrade Rules
- **Quarterly plan gating:** Quarterly option shown only when `QUARTERLY_PLAN_ENABLED=true` (env flag, TERMINOLOGY.md §6)
- **Conversion discount:** "Show any first-invoice discount from unused purchased pack value" — implies display of credit conversion amount if applicable (story scope, unclear implementation detail)
- **Provider-enabled only:** Hidden entirely when `BILLING_ENABLED=false`
- **Two offers:** `unlimited_quarterly` (if enabled), `unlimited_yearly` (always enabled)

### User-Facing Copy (Frozen)
- Paywall zero-balance prompt: **"Out of explorations. Explore more songs."** (story §Paywall)
- Unlock action: **"Unlock"** (user-facing verb, TERMINOLOGY.md §1)
- CTA copy: **"Unlock more songs"** (user-facing, TERMINOLOGY.md §2)
- Balance unit: internal **"credit"**, user-facing **"songs to explore"** (TERMINOLOGY.md §3)

### Cache Invalidation Timing
- **After checkout return:** Invalidate `billingState`, `likedSongsKeys.all`, `likedSongsKeys.stats(accountId)` (story scope)
- **After portal return:** Invalidate `billingState` only (story scope)
- **After successful unlock:** Invalidate `billingState`, `likedSongsKeys.all`, `likedSongsKeys.stats(accountId)` (story scope, already implemented in `useSongUnlock`)
- **Ensure useActiveJobs handles billing-triggered job completions:** If balance change triggers enrichment queue changes, `useActiveJobCompletionEffects` must react (story scope, constraint unclear)

### Provider-Disabled Mode
- When `BILLING_ENABLED=false`:
  - All checkout/portal CTAs hidden
  - Paywall shown for zero balance (but no upgrade path visible)
  - Users have `unlimitedAccess.kind === "self_hosted"` (self-healing in account provisioning)
  - Display label: **"Unlimited"** (TERMINOLOGY.md §2)

## Summary

S6-05 integrates post-purchase flows and cache invalidation. **The foundational infrastructure is in place:**
- `createCheckoutSession` and `createPortalSession` server functions exist and are provider-aware
- Query key structure (likedSongsKeys, dashboardKeys, billingQueryKey) is defined and accessible
- `useSongUnlock` hook already demonstrates the invalidation pattern
- `useActiveJobCompletionEffects` already invalidates queries on job completion
- Insufficient balance error state exists in UnlockConfirmDialog

**The implementation agent must:**
1. Create a paywall component (or extend UnlockConfirmDialog) with pack and unlimited CTAs
2. Add checkout/portal return route handling (with Stripe session_id parsing, optional)
3. Wire paywall display logic into liked songs page when balance is zero
4. Create/extend checkout and portal return routes to trigger cache invalidation
5. Ensure pack CTA is hidden when `hasUnlimitedAccess(billingState) === true`
6. Ensure all CTAs are hidden when `BILLING_ENABLED=false`
7. Add tests covering all flows

**Key unknowns:** exact paywall UX, checkout/portal return routes, conversion discount calculation, and whether billing state changes require separate listener in `useActiveJobs`.
