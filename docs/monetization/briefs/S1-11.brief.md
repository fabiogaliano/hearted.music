# Implementation Brief: S1-11 Billing-Aware Selector RPCs

## Goal

Implement two billing-aware selector functions that replace the ungated
`select_liked_song_ids_needing_pipeline_processing` and
`select_data_enriched_liked_song_ids` selectors for billing-scoped work:

- `select_liked_song_ids_needing_enrichment_work` — enrichment work selector with per-stage flags
- `select_entitled_data_enriched_liked_song_ids` — match-refresh candidate selector with entitlement gate

-- Full-pipeline selector (ungated — no entitlement filter)
CREATE OR REPLACE FUNCTION select_liked_song_ids_needing_pipeline_processing(
  p_account_id UUID,
  p_limit INTEGER
)
RETURNS TABLE(song_id UUID)
LANGUAGE sql
STABLE
SECURITY DEFINER
AS $$
  SELECT ls.song_id
  FROM liked_song ls
  INNER JOIN song s ON s.id = ls.song_id
  WHERE ls.account_id = p_account_id
    AND ls.unliked_at IS NULL
    AND NOT EXISTS (
      SELECT 1 FROM job_failure jf
      INNER JOIN job j ON j.id = jf.job_id
      WHERE jf.item_id = ls.song_id
        AND jf.item_type = 'song'
        AND jf.error_type IN ('validation', 'unsupported', 'auth', 'permanent')
        AND j.account_id = p_account_id
    )
    AND (
      NOT EXISTS (SELECT 1 FROM song_audio_feature saf WHERE saf.song_id = ls.song_id)
      OR s.genres IS NULL OR array_length(s.genres, 1) IS NULL
      OR NOT EXISTS (SELECT 1 FROM song_analysis sa WHERE sa.song_id = ls.song_id)
      OR NOT EXISTS (SELECT 1 FROM song_embedding se WHERE se.song_id = ls.song_id)
      OR NOT EXISTS (
        SELECT 1 FROM item_status ist
        WHERE ist.account_id = p_account_id
          AND ist.item_type = 'song'
          AND ist.item_id = ls.song_id
      )
    )
  ORDER BY ls.liked_at DESC
  LIMIT p_limit;
$$;

-- Data-enrichment selector (ungated — no entitlement filter)
CREATE OR REPLACE FUNCTION select_data_enriched_liked_song_ids(
  p_account_id UUID
)
RETURNS TABLE(song_id UUID)
LANGUAGE sql
STABLE
SECURITY DEFINER
AS $$
  SELECT ls.song_id
  FROM liked_song ls
  INNER JOIN song s ON s.id = ls.song_id
  WHERE ls.account_id = p_account_id
    AND ls.unliked_at IS NULL
    AND EXISTS (SELECT 1 FROM song_audio_feature saf WHERE saf.song_id = ls.song_id)
    AND s.genres IS NOT NULL AND array_length(s.genres, 1) IS NOT NULL

**Gaps these selectors have:**
- No entitlement filtering — Phase B/C work runs for all liked songs regardless of billing
- `select_liked_song_ids_needing_pipeline_processing` returns a single `song_id` with no stage flags; orchestrator must re-derive work per song
- Missing `item_status` is treated as work for ALL songs, not only entitled ones
- No `SET search_path = public`

Neither function is referenced in TypeScript yet (`grep` returns no matches in `src/`).

### Entitlement predicate (S1-04, already implemented)

**`supabase/migrations/20260405130000_entitlement_predicate.sql`**

```sql
CREATE OR REPLACE FUNCTION is_account_song_entitled(
  p_account_id UUID,
  p_song_id UUID
) RETURNS BOOLEAN
LANGUAGE sql
STABLE
SECURITY DEFINER
SET search_path = public
AS $$
  SELECT (
    EXISTS(
      SELECT 1 FROM account_song_unlock
      WHERE account_id = p_account_id
        AND song_id = p_song_id
        AND revoked_at IS NULL
    )
    OR
    EXISTS(
      SELECT 1 FROM account_billing
      WHERE account_id = p_account_id
        AND unlimited_access_source IS NOT NULL
        AND (
          unlimited_access_source = 'self_hosted'
          OR (unlimited_access_source = 'subscription' AND subscription_status = 'active')
        )
    )
  );
$$;
```

The new selectors must NOT call `is_account_song_entitled()` per row (correlated subquery
overhead at scale). Instead, they compute account-level billing facts once via a CTE and
fold them into the per-row EXISTS checks.

### Relevant schema facts

**`liked_song`** — join source, soft-delete via `unliked_at`
```sql
-- account_id, song_id, liked_at, unliked_at (NULL = active)
-- UNIQUE(account_id, song_id)
-- idx_liked_song_liked_at ON liked_song(account_id, liked_at DESC)
```

**`song`** — `genres TEXT[] NOT NULL DEFAULT '{}'`  
Genre-tagging complete = `array_length(s.genres, 1) IS NOT NULL` (non-empty array).  
Note: `s.genres IS NULL` is never true because of the `NOT NULL DEFAULT '{}'` constraint,
but `array_length(s.genres, 1) IS NULL` is true for the empty-array default.

**`song_audio_feature`** — `UNIQUE(song_id)` — one row = Phase A artifact present

**`song_analysis`** — multiple rows per song allowed; `EXISTS` = analysis present  
Additional columns added in S1-03: `provider`, `input_tokens`, `output_tokens`, `cost_usd`

**`song_embedding`** — `UNIQUE(song_id, kind, model, model_version, content_hash)`;
`EXISTS` = at least one embedding present

**`item_status`** — `UNIQUE(account_id, item_type, item_id)`; `item_type` enum (`'song'|'playlist'`);
`action_type` and `actioned_at` columns removed in migration `20260316050001`

**`account_song_unlock`** — entitlement source; `revoked_at IS NULL` = active unlock
```sql
UNIQUE(account_id, song_id)
-- idx_account_song_unlock_account ON account_song_unlock(account_id)
```

**`account_billing`** — entitlement source; one row per account
```sql
unlimited_access_source TEXT CHECK (... IN ('subscription', 'self_hosted'))
subscription_status     TEXT NOT NULL DEFAULT 'none'
  CHECK (... IN ('none','active','past_due','canceled','unpaid','incomplete','incomplete_expired'))
```

**`job_failure`** — terminal failure detection
```sql
-- error_type TEXT; terminal values: 'validation', 'unsupported', 'auth', 'permanent'
-- job_id FK → job.account_id scopes failures to the account
```

```sql
-- S1-11: Billing-aware enrichment and match-refresh selector RPCs
-- Depends on: billing_core_tables (S1-01), entitlement_predicate (S1-04)
-- These selectors complement (not replace) the legacy ungated selectors;
-- removal of legacy selectors is deferred to Phase 3 orchestrator rewire.

-- Enrichment work selector: returns liked songs that have at least one outstanding
-- enrichment stage, with per-stage flags. Phase A flags (audio_features,
-- genre_tagging) are returned regardless of entitlement. Phase B/C flags
-- (analysis, embedding) and content_activation require effective entitlement.
--
-- Returns TABLE(
--   song_id                 UUID,
--   needs_audio_features    BOOLEAN,
--   needs_genre_tagging     BOOLEAN,
--   needs_analysis          BOOLEAN,
--   needs_embedding         BOOLEAN,
--   needs_content_activation BOOLEAN
-- )
CREATE OR REPLACE FUNCTION select_liked_song_ids_needing_enrichment_work(
  p_account_id UUID,
  p_limit      INTEGER
)
RETURNS TABLE(
  song_id                  UUID,
  needs_audio_features     BOOLEAN,
  needs_genre_tagging      BOOLEAN,
  needs_analysis           BOOLEAN,
  needs_embedding          BOOLEAN,
  needs_content_activation BOOLEAN
)
LANGUAGE sql
STABLE
SECURITY DEFINER
SET search_path = public
AS $$
  WITH billing_facts AS (
    -- Compute unlimited-access status once for the account to avoid per-row
    -- lookups against account_billing.
    SELECT COALESCE(
      (
        SELECT
          unlimited_access_source IS NOT NULL
          AND (
            unlimited_access_source = 'self_hosted'
            OR (
              unlimited_access_source = 'subscription'
              AND subscription_status = 'active'
            )
          )
        FROM account_billing
        WHERE account_id = p_account_id
      ),
      false
    ) AS has_unlimited_access
  )
  SELECT
    ls.song_id,

    -- Phase A: shared artifact missing — no entitlement gate
    NOT EXISTS (
      SELECT 1 FROM song_audio_feature saf WHERE saf.song_id = ls.song_id
    ) AS needs_audio_features,

    (array_length(s.genres, 1) IS NULL)
      AS needs_genre_tagging,

    -- Phase B: entitled AND analysis missing
    (
      (bf.has_unlimited_access OR EXISTS (
        SELECT 1 FROM account_song_unlock asu
        WHERE asu.account_id = p_account_id
          AND asu.song_id = ls.song_id
          AND asu.revoked_at IS NULL
      ))
      AND NOT EXISTS (SELECT 1 FROM song_analysis sa WHERE sa.song_id = ls.song_id)
    ) AS needs_analysis,

    -- Phase C: entitled AND embedding missing
    (
      (bf.has_unlimited_access OR EXISTS (
        SELECT 1 FROM account_song_unlock asu
        WHERE asu.account_id = p_account_id
          AND asu.song_id = ls.song_id
          AND asu.revoked_at IS NULL
      ))
      AND NOT EXISTS (SELECT 1 FROM song_embedding se WHERE se.song_id = ls.song_id)
    ) AS needs_embedding,

    -- Content activation: entitled AND analysis exists AND item_status missing
    (
      (bf.has_unlimited_access OR EXISTS (
        SELECT 1 FROM account_song_unlock asu
        WHERE asu.account_id = p_account_id
          AND asu.song_id = ls.song_id
          AND asu.revoked_at IS NULL
      ))
      AND EXISTS (SELECT 1 FROM song_analysis sa WHERE sa.song_id = ls.song_id)
      AND NOT EXISTS (
        SELECT 1 FROM item_status ist
        WHERE ist.account_id = p_account_id
          AND ist.item_type = 'song'
          AND ist.item_id = ls.song_id
      )
    ) AS needs_content_activation

  FROM liked_song ls
  CROSS JOIN billing_facts bf
  INNER JOIN song s ON s.id = ls.song_id

  WHERE ls.account_id = p_account_id
    AND ls.unliked_at IS NULL

    -- Exclude terminal failures for this account
    AND NOT EXISTS (
      SELECT 1 FROM job_failure jf
      INNER JOIN job j ON j.id = jf.job_id
      WHERE jf.item_id = ls.song_id
        AND jf.item_type = 'song'
        AND jf.error_type IN ('validation', 'unsupported', 'auth', 'permanent')
        AND j.account_id = p_account_id
    )

    -- Return song only when at least one flag would be true:
    -- Phase A work (no entitlement needed)
    AND (
      NOT EXISTS (SELECT 1 FROM song_audio_feature saf WHERE saf.song_id = ls.song_id)
      OR array_length(s.genres, 1) IS NULL
      -- Phase B/C/activation work (entitlement required)
      OR (
        (bf.has_unlimited_access OR EXISTS (
          SELECT 1 FROM account_song_unlock asu
          WHERE asu.account_id = p_account_id
            AND asu.song_id = ls.song_id
            AND asu.revoked_at IS NULL
        ))
        AND (
          NOT EXISTS (SELECT 1 FROM song_analysis sa WHERE sa.song_id = ls.song_id)
          OR NOT EXISTS (SELECT 1 FROM song_embedding se WHERE se.song_id = ls.song_id)
          -- content_activation guard: analysis must exist for this to be the sole reason
          OR (
            EXISTS (SELECT 1 FROM song_analysis sa WHERE sa.song_id = ls.song_id)
            AND NOT EXISTS (
              SELECT 1 FROM item_status ist
              WHERE ist.account_id = p_account_id
                AND ist.item_type = 'song'
                AND ist.item_id = ls.song_id
            )
          )
        )
      )
    )

  ORDER BY ls.liked_at DESC
  LIMIT p_limit;
$$;


-- Match-refresh candidate selector: returns liked songs that have all four shared
-- artifacts AND effective entitlement. Used by the orchestrator to build match
-- snapshot refresh candidate sets. Does NOT require item_status.
CREATE OR REPLACE FUNCTION select_entitled_data_enriched_liked_song_ids(
  p_account_id UUID
)
RETURNS TABLE(song_id UUID)
LANGUAGE sql
STABLE
SECURITY DEFINER
SET search_path = public
AS $$
  WITH billing_facts AS (
    SELECT COALESCE(
      (
        SELECT
          unlimited_access_source IS NOT NULL
          AND (
            unlimited_access_source = 'self_hosted'
            OR (
              unlimited_access_source = 'subscription'
              AND subscription_status = 'active'
            )
          )
        FROM account_billing
        WHERE account_id = p_account_id
      ),
      false
    ) AS has_unlimited_access
  )
  SELECT ls.song_id
  FROM liked_song ls
  CROSS JOIN billing_facts bf
  INNER JOIN song s ON s.id = ls.song_id

  WHERE ls.account_id = p_account_id
    AND ls.unliked_at IS NULL

    -- All four shared artifacts must exist
    AND EXISTS (SELECT 1 FROM song_audio_feature saf WHERE saf.song_id = ls.song_id)
    AND array_length(s.genres, 1) IS NOT NULL
    AND EXISTS (SELECT 1 FROM song_analysis sa WHERE sa.song_id = ls.song_id)
    AND EXISTS (SELECT 1 FROM song_embedding se WHERE se.song_id = ls.song_id)

    -- Effective entitlement: active unlock row OR unlimited access
    AND (
      bf.has_unlimited_access
      OR EXISTS (
        SELECT 1 FROM account_song_unlock asu
        WHERE asu.account_id = p_account_id
          AND asu.song_id = ls.song_id
          AND asu.revoked_at IS NULL
      )
    );
$$;
```

## Flag Semantics Reference

| Flag | Entitlement required | Condition |
|---|---|---|
| `needs_audio_features` | ❌ No | `NOT EXISTS (song_audio_feature)` |
| `needs_genre_tagging` | ❌ No | `array_length(song.genres, 1) IS NULL` |
| `needs_analysis` | ✅ Yes | entitled AND `NOT EXISTS (song_analysis)` |
| `needs_embedding` | ✅ Yes | entitled AND `NOT EXISTS (song_embedding)` |
| `needs_content_activation` | ✅ Yes | entitled AND `EXISTS (song_analysis)` AND `NOT EXISTS (item_status)` |

A song is returned when **any** flag is `true`. Terminal failures are always excluded.

## Acceptance Criteria

- [ ] Enrichment selector returns correct stage flags for: unlocked song missing analysis (`needs_analysis=true`), locked song missing analysis (`needs_analysis=false`), song with all artifacts but missing `item_status` (`needs_content_activation=true` if entitled, `false` if not)
- [ ] Phase A flags (`needs_audio_features`, `needs_genre_tagging`) are `true` regardless of entitlement when artifact is missing
- [ ] Phase B/C flags (`needs_analysis`, `needs_embedding`) are `false` for non-entitled songs even when artifact is missing
- [ ] `needs_content_activation` requires entitlement + analysis exists + `item_status` missing; false if no analysis even when entitled
- [ ] Song NOT returned when only reason would be missing `item_status` on a non-entitled song
- [ ] Song IS returned for a non-entitled song missing audio features or genres
- [ ] Match-refresh selector returns only entitled + fully-enriched (all 4 artifacts) songs
- [ ] Revoked songs (`revoked_at IS NOT NULL` with no unlimited access) excluded from both selectors
- [ ] `self_hosted` accounts see all their liked songs as entitled in both selectors
- [ ] Terminal failures (`error_type IN ('validation', 'unsupported', 'auth', 'permanent')`) excluded from enrichment selector
- [ ] Enrichment selector respects `p_limit` and orders by `liked_at DESC`
- [ ] Both functions are `STABLE`, `SECURITY DEFINER`, `SET search_path = public`
- [ ] `supabase db reset` completes cleanly

## Risks and Ambiguities

### 1. `billing_facts` CTE with CROSS JOIN — empty `account_billing` row

If an `account_billing` row doesn't exist for the account (invariant violation per DECISIONS.md §3),
the COALESCE in the CTE ensures `has_unlimited_access = false` and the CTE still returns exactly one
row. The CROSS JOIN therefore never filters out songs due to missing billing state — Phase A work
is always returned correctly even for misconfigured accounts.

### 2. Per-song unlock EXISTS repeated in SELECT and WHERE

The per-song unlock `EXISTS (account_song_unlock WHERE revoked_at IS NULL)` appears in both
the SELECT (for flag computation) and the WHERE (for filtering). PostgreSQL's planner should
deduplicate identical subqueries within the same query block, but this is not guaranteed. If
query performance on large liked-song libraries becomes an issue, a lateral subquery or CTE
that materializes entitlement per song could be considered — but that's a future optimization,
not a correctness concern for v1.

### 3. `needs_content_activation` guard in WHERE clause

The WHERE clause includes:
```sql
EXISTS (song_analysis) AND NOT EXISTS (item_status)
```
for the content-activation guard. This is stricter than just `NOT EXISTS (item_status)`.
The reason for the `EXISTS (song_analysis)` guard: without it, a non-entitled song missing
item_status (and also lacking analysis) would be admitted to the entitlement block only
because `NOT EXISTS (item_status)` matches — but `needs_analysis` and `needs_embedding`
flags would be false (non-entitled), and `needs_content_activation` would also be false
(no analysis). The song would be returned with all five flags false, which violates the
"song returned when ANY flag is true" invariant. Adding the analysis guard prevents this
case. (The `NOT EXISTS (song_analysis)` arm in the entitlement block already handles songs
missing analysis, so there's no exclusion of legitimate work.)

### 4. `song.genres` array length check

The `song.genres` column is `TEXT[] NOT NULL DEFAULT '{}'`. The check `array_length(s.genres, 1) IS NULL`
correctly identifies empty arrays (returns NULL for empty arrays, not 0). The legacy selector also
checked `s.genres IS NULL` which can never be true given the NOT NULL constraint — this dead
condition is intentionally dropped in the new selectors.

### 5. Old selectors not removed

Per story scope: "Removing old selectors (done when orchestrator is rewired in Phase 3)". The
new selectors coexist with `select_liked_song_ids_needing_pipeline_processing` and
`select_data_enriched_liked_song_ids` until Phase 3. There is no TypeScript consumer of either
the old or new selectors yet.

### 6. `select_entitled_data_enriched_liked_song_ids` omits terminal failure exclusion

The legacy `select_data_enriched_liked_song_ids` also did not exclude terminal failures
("refresh eligibility is separate from pipeline work" per the original migration comment). This
story does not change that — the match-refresh selector intentionally omits terminal failure
exclusion for the same reason. The story spec for this function makes no mention of terminal
failure exclusion.
