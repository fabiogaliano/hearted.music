function filterBatch(batch: PipelineBatch, songIds: string[]): PipelineBatch {
  const idSet = new Set(songIds);
  return {
    songIds: batch.songIds.filter((id) => idSet.has(id)),
    songs: batch.songs.filter((s) => idSet.has(s.id)),
    spotifyIdBySongId: new Map(
      [...batch.spotifyIdBySongId].filter(([id]) => idSet.has(id)),
    ),
  };
}

**D. Rewrite `enrichSongs`** to accept `EnrichmentWorkPlan` + full `PipelineBatch`, dispatch per sub-batch:

```typescript
async function enrichSongs(
  ctx: EnrichmentContext,
  workPlan: EnrichmentWorkPlan,
  batch: PipelineBatch,
  jobId: string,
  progress: InitializedEnrichmentChunkProgress,
  stageDelayMs?: number,
): Promise<void> {
  // Phase A: audio_features + genre_tagging (unbounded — all songs)
  progress.currentStage = "audio_features";
  progress.stages.audio_features.status = "running";
  progress.stages.genre_tagging.status = "running";
  await persistProgress(jobId, progress);

  const audioSubBatch = filterBatch(batch, workPlan.needAudioFeatures);
  const genreSubBatch = filterBatch(batch, workPlan.needGenreTagging);

  const [audioResult, genreResult] = await Promise.all([
    audioSubBatch.songIds.length > 0
      ? runStage("audio_features", () => runAudioFeatures(ctx, audioSubBatch))
      : Promise.resolve({ total: 0, succeeded: 0, failed: 0 }),
    genreSubBatch.songIds.length > 0
      ? runStage("genre_tagging", () => runGenreTagging(ctx, genreSubBatch))
      : Promise.resolve({ total: 0, succeeded: 0, failed: 0 }),
  ]);

  applyStageResult(
    progress, "audio_features", audioResult,
    audioSubBatch.songIds.length > 0 ? stageStatus(audioResult) : "skipped",
  );
  applyStageResult(
    progress, "genre_tagging", genreResult,
    genreSubBatch.songIds.length > 0 ? stageStatus(genreResult) : "skipped",
  );

  // Phase B: song_analysis (entitled only)
  await maybeDevDelay(stageDelayMs);
  progress.currentStage = "song_analysis";
  progress.stages.song_analysis.status = "running";
  await persistProgress(jobId, progress);

  const analysisSubBatch = filterBatch(batch, workPlan.needAnalysis);
  const analysisResult =
    analysisSubBatch.songIds.length > 0
      ? await runStage("song_analysis", () => runSongAnalysis(ctx, analysisSubBatch))
      : { total: 0, succeeded: 0, failed: 0 };

  applyStageResult(
    progress, "song_analysis", analysisResult,
    analysisSubBatch.songIds.length > 0 ? stageStatus(analysisResult) : "skipped",
  );

  // Phase C: song_embedding (entitled only)
  await maybeDevDelay(stageDelayMs);
  progress.currentStage = "song_embedding";
  progress.stages.song_embedding.status = "running";
  await persistProgress(jobId, progress);

  const embeddingSubBatch = filterBatch(batch, workPlan.needEmbedding);
  const embeddingResult =
    embeddingSubBatch.songIds.length > 0
      ? await runStage("song_embedding", () => runSongEmbedding(ctx, embeddingSubBatch))
      : { total: 0, succeeded: 0, failed: 0 };

  applyStageResult(
    progress, "song_embedding", embeddingResult,
    embeddingSubBatch.songIds.length > 0 ? stageStatus(embeddingResult) : "skipped",
  );

  // Content activation placeholder (S3-03)
  // TODO(S3-03): runContentActivation(ctx, workPlan.needContentActivation)
}
```

## Constraints

From **DECISIONS.md §5 — Workflow Invariants**:

> "No second durable workflow for Phase A vs Phase B/C. The orchestrator uses per-song stage flags and runs each stage against the exact sub-batch that needs it."

Pipeline split:
- Phase A (`audio_features`, `genre_tagging`) — unbounded, no billing gate
- Phase B (`song_analysis`) — gated by effective entitlement
- Phase C (`song_embedding`) — gated by effective entitlement

From **DECISIONS.md §6 — Terminology Invariants**:

> Selector flags frozen: `needs_audio_features`, `needs_genre_tagging`, `needs_analysis`, `needs_embedding`, `needs_content_activation`

From **CLAUDE.md**:
- No barrel exports
- No `any`, no non-null assertions (`!`), no unsafe casts

**Stage runner signatures are frozen** — `runAudioFeatures`, `runGenreTagging`, `runSongAnalysis`, `runSongEmbedding` accept `(ctx, batch: PipelineBatch)` and must not be changed.

`markPipelineProcessed` call stays — removal is S3-04 scope.

Progress accounting (`makeInitialProgress`, total calculation) stays as-is — S3-04 scope.

## Acceptance Criteria

- [ ] **Phase A stages run for songs regardless of entitlement** — `runAudioFeatures` / `runGenreTagging` dispatched with `workPlan.needAudioFeatures` / `workPlan.needGenreTagging` sub-batches
- [ ] **Phase B/C stages run only for entitled songs** — `runSongAnalysis` uses `workPlan.needAnalysis`; `runSongEmbedding` uses `workPlan.needEmbedding`
- [ ] **Songs needing only Phase A do not get analysis or embedding** — stage runners not called for empty sub-batches; `"skipped"` status applied
- [ ] **Candidate eligibility snapshot taken before and after stages** — `loadDataEnrichedSongIds` called pre- and post-`enrichSongs`
- [ ] **Stage sub-batches derived from work plan** — no hardcoded IDs; filtered from `workPlan.need*` arrays
- [ ] **Existing stage runner interfaces unchanged** — `runAudioFeatures`, `runGenreTagging`, `runSongAnalysis`, `runSongEmbedding` signatures untouched
- [ ] **Content activation placeholder present** — `TODO(S3-03)` comment after Phase C in `enrichSongs`

## Risks

1. **`loadBatchSongs` filters out songs without a Spotify ID** — the function only adds songs with `song.spotify_id` to `validSongIds`. So `batch.songIds` may be smaller than `workPlan.allSongIds`. `filterBatch` uses set intersection, so work-plan IDs not in `batch.songIds` are silently dropped. This matches existing behavior with `selectPipelineBatch` (same filtering). No behavioral regression.

2. **`makeInitialProgress` total count uses `batch.songIds.length * 4`** — still overestimates for Phase-A-only songs. Intentional deferral to S3-04.

3. **Hot file coordination** — `orchestrator.ts` is also touched by S3-03 (content activation) and S3-04 (progress + `markPipelineProcessed` removal). This story must merge before S3-03. The `TODO(S3-03)` comment is the integration point.

4. **`PipelineBatch` import in `loadDataEnrichedSongIds`** — after removing the `selectPipelineBatch` import, the `Awaited<ReturnType<typeof selectPipelineBatch>>` type reference must be replaced with `PipelineBatch` (imported from `./batch`). Verify no lingering inferred type references.
