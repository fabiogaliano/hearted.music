# S4-10 · createCheckoutSession + createPortalSession Server Functions

## Goal

Implement the app-side server functions that bridge checkout and portal requests to the billing service.

## Why

The public app needs TanStack Start server functions that sign and forward checkout/portal requests to the billing service. These are the only app-side code that talks to the billing service directly.

## Depends on

- S4-01 (HMAC auth pattern for signing)
- S2-01 (env config: `BILLING_ENABLED`, `BILLING_SERVICE_URL`, `BILLING_SHARED_SECRET`)

## Blocks

- Phase 5 (onboarding checkout flow)
- Phase 6 (in-app purchase flows)

## Scope

### `createCheckoutSession({ offer, checkoutAttemptId })`
- Validates `offer` is a valid internal offer ID
- Signs request body with HMAC (shared secret, timestamp, body hash)
- POSTs to billing service `/checkout/pack` or `/checkout/unlimited`
- Returns `{ checkoutUrl }` or typed error
- Fails fast with typed unsupported-mode error when `BILLING_ENABLED=false`

### `createPortalSession()`
- Signs request body with HMAC
- POSTs to billing service `/portal/session`
- Returns `{ portalUrl }` or typed error
- Fails fast when `BILLING_ENABLED=false`

### HMAC signing utility
- Shared utility for signing outbound requests to billing service
- Timestamp + raw body hash + shared secret
- Can live in `src/lib/domains/billing/` or a small platform utility

## Out of scope

- Billing service endpoints (S4-02–S4-04)
- UI that calls these functions (Phase 5, 6)
- Bridge ingress (S4-08)

| Area | Files |

## Constraints / decisions to honor

- `checkoutAttemptId` generated by the app per checkout intent; included in signed body
- Only `v1_hearted_brand/` knows Stripe price IDs — app sends internal offer IDs
- Provider-disabled guard: typed error, not thrown exception

## Acceptance criteria

- [ ] `createCheckoutSession` returns checkout URL for valid offers
- [ ] `createPortalSession` returns portal URL
- [ ] Both fail fast with typed error when `BILLING_ENABLED=false`
- [ ] Requests are HMAC-signed with timestamp and body hash
- [ ] Invalid offer IDs rejected before calling billing service
