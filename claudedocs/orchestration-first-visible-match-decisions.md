This file records decisions made during execution that were **not** spelled out in the plan,
plus a one-line rationale each. Appended to by every subagent and the orchestrator.

## Orchestration-level decisions

- **Phase 0 tests folded into their implementation phases** — The plan's Phase 0 mixes
  characterization tests with some "expected to fail until fixed" tests. To avoid committing a
  red suite, each phase's new tests are written/committed together with the implementation that
  makes them pass, rather than as a separate up-front red commit.
- **Phase 6 treated as a constraint/verification, not a code change** — It explicitly says to keep
  the existing complete-atomic snapshot path. No edits; verified during the Phase 5 review.
- **Phase 10 (scoped/delta snapshots) is out of scope** — Plan marks it optional/later and "should
  not be first."
- **DB migrations are created as files only, not applied to any database** — The orchestrator will
  not autonomously apply migrations to local or prod. The Phase 5 migration file is authored and
  TS code is written to work with it; applying it is left to the user.
